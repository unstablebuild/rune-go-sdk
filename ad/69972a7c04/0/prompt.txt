The querynode and searchnode subcommands in cmd/runectl take a node type argument. This node type is a bitflag, so we should be able to parse any combination of node type strings ORed together: scope|func should result in querying for the bit flag syntaxapi.NodeCaptureScope|syntaxapi.NodeCaptureDefinitionFunc, for example. Currently we expect only one of the node capture names. We should change that. Verify that it works by adding a test. This is a good opportunity for practicing TDD.
