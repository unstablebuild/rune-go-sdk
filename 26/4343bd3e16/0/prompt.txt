# File Explorer Component Plan

## Context

We need a `fileexplorer.Component` (in `component/fileexplorer/`) that
renders a file tree like oil.nvim. It implements `component.Floating`,
uses `[][]term.Cell` as its render buffer, and supports expand/collapse,
node lookup by coordinates, external cell editing with change detection,
and filesystem reload. The root directory contents are loaded via
`workspaceapi.FileSystem.ReadDir`.

## File Structure

```
component/fileexplorer/
  fileexplorer.go       -- Component, Config, constructors, Floating
  tree.go               -- node type, tree ops, render, parse
  operation.go          -- Operation type, diff/changes logic
  fileexplorer_test.go  -- all tests (mock FS + comptest)
```

## Types

### Config (`fileexplorer.go`)

```go

    IndentRune rune            // default: '│' (U+2502)

### Component (`fileexplorer.go`)

    fs     workspaceapi.FileSystem

```

### node (`tree.go`)

```go
type node struct {
    name     string
    uri      workspaceapi.URI
    isDir    bool
    expanded bool
    depth    int
    parent   *node
    children []*node
}
```

### Operation (`operation.go`)

```go
type OperationType int
const (
    OpCreate OperationType = iota
    OpMkdir
    OpDelete
    OpRename
)
type Operation struct {
    Type   OperationType
    URI    workspaceapi.URI // source (delete, rename src)
    NewURI workspaceapi.URI // destination (create, mkdir, rename dst)
}
```

## Rendering Format

Each visible node renders as one line:

```
(<indentRune> ){depth}<icon> <name>[/]
```

Example with `IndentRune='│'`, dir icon ``, go icon ``:
```
 src/
│  main.go
│  utils.go
│  handlers/
│ │  auth.go
 README.md
```

- Root node itself is NOT rendered, only its children
- Dirs end with `/`, files do not
- Dirs sorted before files, alphabetical within each group
- Icon lookup: `cfg.Icons["/"]` for dirs, `cfg.Icons[ext]` for
  files, `cfg.Icons[""]` as fallback, `' '` if nothing configured

## Public API

### Constructors

- `New(fs, root) (*Component, error)` — default config
- `NewWithConfig(fs, root, cfg) (*Component, error)` — custom config

Both: read root children, sort, flatten, render cells.

### component.Floating

- `Resize(width, height int)` — store clip dimensions
- `Draw(w term.Writer)` — draw cells clipped to width/height
- `Dimensions() (int, int)` — optimal width/height of cells

### Tree Navigation

- `NodeAt(pos) (URI, bool)` — URI at coordinates (y→flat index)
- `ExpandNodeAt(pos) (URI, bool)` — file→(uri,true);
  dir→toggle expand, rebuild, return (URI{},false)
- `ExpandLevel(pos)` — expand all dirs at same depth as node at pos
- `CollapseLevel(pos)` — collapse all dirs at same depth
- `CollapseAll()` — collapse everything

### Cell Access

- `Cells() [][]term.Cell` — get buffer for external editing
- `SetCells([][]term.Cell)` — set edited buffer back

### Change Detection

- `Changes() []Operation` — diff edited cells vs internal tree

### Reload

- `Init() error` — re-read FS, rebuild tree, preserve expanded state

## Key Implementation Details

### tree.go functions

1. `readChildren(fs, n *node) error` — calls `fs.ReadDir(n.uri.Path())`,
   creates child nodes with `workspaceapi.Join(n.uri, name)`,
   sets depth/parent, then `sortChildren`.
2. `sortChildren(children []*node)` — dirs first, alpha within groups.
3. `flatten(root *node) []*node` — DFS pre-order, skip collapsed
   children. Root not included.
4. `renderCells(flat, cfg, iconFor) [][]term.Cell` — for each node,
   build line string, use `term.StringToCells` to convert.
5. `parseCells(cells, cfg) []parsedEntry` — for each row, convert
   to string, count indent pairs, skip icon+space, extract name,
   detect trailing `/` for dirs.
6. `collectExpandedPaths(root) map[string]bool` — walk tree,
   collect URIs of expanded dirs.
7. `restoreExpanded(fs, root, expanded) error` — re-expand dirs
   that were expanded before, reading their children.

### operation.go diff algorithm (Changes)

1. Parse edited cells → `[]parsedEntry`
2. Rebuild virtual tree from parsed entries (depth-based parent stack)
3. Per expanded directory, compare old children names vs new:
   - Missing → `OpDelete`
   - Added (with `/`) → `OpMkdir`
   - Added (without `/`) → `OpCreate`
   - Exactly 1 removed + 1 added in same parent → `OpRename`
4. Collect and return all operations

### Lazy loading

Children are read from FS only when first expanded (or at
construction for root). This avoids reading the entire tree.

### rebuild helper

`c.rebuild()` calls `flatten(c.tree)` → `renderCells(...)` → stores
in `c.flat` and `c.cells`.
